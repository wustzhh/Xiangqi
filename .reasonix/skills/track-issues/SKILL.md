# track-issues — 问题追踪 skill

## 行为规则

每次用户描述问题、bug、或新需求时，执行以下流程：

### 1. 检查上一个问题状态
在响应用户新问题/需求之前，先问一句：
> "上一个问题（xxx）是否已修复？"

除非用户明确说「没修复」「还有问题」「没解决」，否则视为已解决，在 issues.md 中将对应条目标记为 ✅ 已修复，并追加一行修复方案说明。

### 2. 记录新问题
在 `docs/issues.md` 中追加新条目：

```markdown
## N. 问题标题

**描述**：用户的原话或总结

**状态**：❌ 未完成
```

如果文件不存在则创建，在文件顶部加标题 `# 待修复问题清单`。

### 3. 修复后记录方案
对问题做出代码修复后，在 issues.md 中更新对应条目：

```markdown
**状态**：✅ 已修复
**修复方案**：简要说明改了哪些文件、怎么修的
```

### 4. 不要删除旧记录
已修复的问题保留在文件中，不要删除。只更新状态标记和追加修复说明。

### 5. 每次对话结束时
检查 issues.md 中有没有标记为「未完成」的条目，如果有，在总结中提示用户还有未解决的问题。
