# 中国象棋手机 App — 项目规划书

## 一、技术选型

| 层 | 技术 |
|------|------|
| UI 框架 | **Flutter**（Android + iOS 双平台） |
| 语言 | Dart |
| 象棋规则引擎 | Dart 纯实现（走法生成、将军/将杀判定） |
| AI 引擎 | Dart 实现 Minimax + Alpha-Beta 剪枝 |
| 音效 | `audioplayers` 包 |
| 本地存储 | `shared_preferences` / 文件 JSON |
| 网络对战 | WebSocket（客户端 `web_socket_channel`，服务端待定） |
| 服务器 | 可选 Dart `shelf` / Python `websockets` / Node.js |

## 三、棋盘数据模型（核心约定）

棋盘用 **9 列 × 10 行** 二维数组表示：

```
  列:  0  1  2  3  4  5  6  7  8    (红方视角，列从左到右)
行 0: [車][馬][相][仕][帥][仕][相][馬][車]   ← 红方底线
行 1: [  ][  ][  ][  ][  ][  ][  ][  ][  ]
行 2: [  ][炮][  ][  ][  ][  ][  ][炮][  ]
行 3: [兵][  ][兵][  ][兵][  ][兵][  ][兵]
行 4: [  ][  ][  ][  ][  ][  ][  ][  ][  ]   ← 楚河
────────────────────────────────────────
行 5: [  ][  ][  ][  ][  ][  ][  ][  ][  ]   ← 汉界
行 6: [卒][  ][卒][  ][卒][  ][卒][  ][卒]
行 7: [  ][砲][  ][  ][  ][  ][  ][砲][  ]
行 8: [  ][  ][  ][  ][  ][  ][  ][  ][  ]
行 9: [車][馬][象][仕][將][仕][象][馬][車]   ← 黑方底线
```

**坐标约定**：`(col, row)` — `col 0-8, row 0-9`
- 红方在下方（row 5-9），黑方在上方（row 0-4）
- `board[row][col]` 取值：`null`（空）或 `Piece` 对象

## 四、开发路线（四个阶段）

### 第一阶段：显示与交互 + 调试（优先级最高）

**目标**：可运行 App，看到棋盘棋子，能点棋子走棋，有调试面板

| 子任务 | 内容 |
|--------|------|
| 1.1 项目初始化 | Flutter 项目脚手架、pubspec.yaml 依赖配置 |
| 1.2 棋盘绘制 | `CustomPainter` 画棋盘网格、楚河汉界、坐标标注 |
| 1.3 棋子绘制 | 画圆形棋子 + 文字（红黑两色），支持选中高亮 |
| 1.4 棋子数据模型 | `Piece` 枚举、`Board` 数组、初始局面 |
| 1.5 点击走棋 | 点击选中 → 点击目标走棋（先无规则限制，纯自由移动） |
| 1.6 规则引擎 | 走法合法性校验 + 将军判定 + 应将/将杀 |
| 1.7 走棋动画 | 棋子移动平行动画、吃子消失动画 |
| 1.8 音效 | 走棋/吃子/将军音效播放 |
| 1.9 调试面板 | 覆盖层显示：棋盘数组、当前玩家、将军状态、走法列表 |
| 1.10 游戏状态机 | 完整回合流转（红走→黑走→红走…） |

**产出**：双人对弈可用，调试面板可查看内部状态

### 第二阶段：单机模式 + 难度选择

**目标**：可对战 AI，可选难度

| 子任务 | 内容 |
|--------|------|
| 2.1 局面评估函数 | 子力价值 + 位置价值表 |
| 2.2 AI 搜索 | Minimax + Alpha-Beta 剪枝，深度可控 |
| 2.3 难度封装 | 初级（2层）、中级（3层）、高级（4层+） |
| 2.4 AI 异步计算 | Isolate 后台计算，不卡 UI |
| 2.5 主菜单 UI | 开始游戏、设置、关于 |
| 2.6 难度选择 UI | 选红/黑、选难度 |
| 2.7 游戏结束处理 | 将杀/困毙判定、胜负弹窗、重新开始 |

**产出**：完整人机对战，三个难度可选

### 第三阶段：对局记录 + 复盘 + 中途模拟

**目标**：能存盘、回顾、分析局面

| 子任务 | 内容 |
|--------|------|
| 3.1 走法记录 | 每步记录为 `Move` 对象（起止坐标、棋子、吃子） |
| 3.2 对局序列化 | 走法列表 ↔ JSON，含时间戳、结果 |
| 3.3 本地存储 | 存档列表、读取、删除 |
| 3.4 复盘播放器 | 前进/后退/跳转步数、自动播放 |
| 3.5 中途模拟分支 | 从任意步数分叉，进入模拟走棋（不影响原记录） |
| 3.6 对局列表 UI | 历史存档列表浏览 |
| 3.7 复盘 UI | 棋盘 + 走法列表 + 播放控制条 |
| **3.8 棋子统计** | 显示双方被吃棋子列表、被护住棋子数量 |
| **3.9 攻击提示** | 高亮显示敌方可以吃到的己方棋子位置 |

**产出**：完整复盘系统，可分支模拟，带局面分析功能

### 🎯 附加功能：局面分析工具（对局中实时分析）

| 子任务 | 内容 |
|--------|------|
| A.1 护子模式 | 显示每个友军被多少棋子护住；点击棋子高亮显示护住它的具体棋子（绿色大炮图标） |
| A.2 攻击模式 | 用红色X标记敌方所有可攻击到的格子和棋子；选中己方棋子时只显示该棋子的可吃目标 |
| A.3 安全模式 | 安全分 = 护子数 - 被攻数，7档着色（+3~-3）。友方：绿安全/红危险。**敌方：颜色反转** |
| A.4 危险格模式 | 每格被多少敌方棋子攻击，7档着色（6+红→4橙→2黄→1浅黄） |
| A.5 按钮行为 | 四个按钮互斥切换；点同一按钮取消；点其他按钮切换；对局结束时隐藏 |

**产出**：四种棋盘分析覆盖层，帮助玩家分析局面

### 第四阶段：网络联机

**目标**：小型网络对战（≤10 人同时在线）

| 子任务 | 内容 |
|--------|------|
| 4.1 通信协议设计 | WebSocket 消息格式定义（JSON） |
| 4.2 服务端基础架子 | 房间管理、连接保活 |
| 4.3 对局会话 | 双人走棋同步、计时 |
| 4.4 客户端网络层 | WebSocket 连接、重连、消息分发 |
| 4.5 房间 UI | 创建/加入房间、等待匹配 |
| 4.6 联机对局 UI | 走棋同步、对手信息、断开处理 |
| 4.7 服务器部署文档 | 部署脚本、配置说明 |

**产出**：可联机对战，支持 5 桌同时游戏

## 五、关键设计决策

| 决策 | 选项 | 选定 |
|------|------|------|
| 状态管理 | Provider / Riverpod / Bloc | **Provider**（轻量、够用） |
| 棋盘绘制 | CustomPainter / 图片拼接 | **CustomPainter**（高清、自适应） |
| 棋盘背景 | 纯渐变色 / 木纹图 | **木纹图**（CC0 无缝纹理） |
| 棋子样式 | 纯文字 / 图片 / 纹理 | **纯文字圆形**，CustomPainter 径向渐变模拟木纹质感 |
| 棋子文字 | 简体 / 繁体 | **简体**默认，设置中可切换繁体 |
| 走法记谱 | 中国坐标记谱 | 参考象棋竞赛规则 |
| AI 搜索 | Minimax / MCTS | **Minimax + Alpha-Beta**（稳定可控） |
| 网络协议 | REST / WebSocket / gRPC | **WebSocket**（实时双向，轻量） |
| 本地存档 | JSON 文件 / SQLite | **JSON 文件**（简单、可读、易调试） |

## 六、名词定义

| 术语 | 含义 |
|------|------|
| `Piece` | 棋子，含类型（PieceType）和颜色（Side） |
| `Side` | 红方/黑方 |
| `Move` | 一步走法，含起点、终点、走棋方、吃子信息 |
| `Board` | 9×10 棋盘数组 |
| `GameState` | 完整对局状态（Board + 走法历史 + 当前玩家 + 状态） |
| `GameRecord` | 完整的对局记录（元信息 + Move 列表） |
| `Node` | 棋盘上的一个位置 (col, row) |
